# Engineering Practicum: Project Manifest

An engineered, deterministic multi-agent workflow engine that automates the full SDLC
without human intervention. This document is the high-level manifest and the distilled
engineering record. The chronological development history — release-by-release — now
lives in [CHANGELOG.md](./CHANGELOG.md); the decisions behind each iteration are recorded
as Architecture Decision Records in [docs/adr/](./docs/adr/).

## Key Engineering Takeaways
* **Prompt Engineering Belongs in Prompts, Not in Schema Definitions** *([ADR 0007](./docs/adr/0007-prompt-schema-layer-separation.md))*: When behavioral LLM directives live inside Pydantic `Field(description=...)` strings, tuning an agent requires editing Python infrastructure code and redeploying the core package — and the same rule exists in two places (schema + system prompt) with no enforcement of consistency. The fix is a strict layer boundary: schema `Field` descriptions carry only dry structural intent ("Boolean flag indicating approval status"); all behavioral rules ("Set to `true` ONLY IF tests are written without loopholes") migrate to a `### Output JSON Schema Semantics` section in the agent's system prompt file, bound per JSON key. An `lru_cache`d prompt loader resolves markdown files from `prompts/system/` at runtime, making behavior editable without touching application code. Iteration-7 validation across the three reference contracts (Prime 1 cycle / 32 tests, Fibonacci 1 cycle / 16 tests, Geometry 3 cycles / 70 tests, all zero-SAST) found **zero system-prompt leakage into the sandbox** (`artifacts/code/`), confirming the boundary holds at runtime. The generalizable rule: infrastructure schemas describe shape; system prompts describe judgment.

* **Transactional Integrity Across Ephemeral Runtimes** *([ADR 0006](./docs/adr/0006-fsm-state-serialization-resume.md))*: Any pipeline whose state lives only in a Python process is one `Ctrl+C` away from full token loss. The fix is a single rolling `checkpoint.json` written by `GlobalPipelineContext.save_checkpoint` after every critical node, plus a `--resume` CLI path that loads it through `model_validate_json` and **bypasses** any node whose output already exists in the restored context. The Circuit Breaker counter (`current_attempt`) must be promoted from a loop variable to a persisted Pydantic field — otherwise process restarts silently reset the retry budget and defeat the safety guarantee. A companion `--reset-attempts` flag covers the canonical "I fixed the prompt, retry from scratch but keep the contract" workflow. The generalizable rule: every piece of ephemeral runtime state that gates routing decisions (e.g. `regenerate_tests`) must be **derivable from persisted fields** on resume — never reconstructed from in-memory flags.

* **Confirmation Bias as a Token Tax** *([ADR 0006](./docs/adr/0006-fsm-state-serialization-resume.md))*: When the Developer agent (Claude CLI) was free to "helpfully" emit unit tests next to production modules, those tests polluted the `production_code_snapshot` and effectively let the Developer grade its own work — a textbook confirmation-bias loop that wasted QA tokens regenerating already-passing assertions. Symmetrically, QA tests that asserted on exception strings (`assertIn("must be positive", str(exc))`) tightly coupled the test suite to the Developer's wording, producing false negatives on any prompt rewording. Resolution: hermetic prompt-level prohibitions — Developer is explicitly forbidden from writing tests; QA is explicitly forbidden from inspecting exception strings. Scope isolation must be enforced at the LLM boundary, not by post-hoc filtering of agent output.

* **Architectural Invariants Belong in the Architect Prompt** *([ADR 0006](./docs/adr/0006-fsm-state-serialization-resume.md))*: Tight Coupling (e.g. `Cylinder` instantiating `Circle()` inside its body) is an architectural defect that the Reviewer can detect only after tokens are spent on the wrong code. Encoding the invariant — "Classes must receive their dependencies via the constructor; STRICTLY FORBIDDEN from instantiating dependencies internally" — directly into the Architect's system prompt blocks the antipattern at the contract layer. Per-agent prompts are the cheapest enforcement boundary for cross-cutting design rules.

* **Git Anchor as a State Machine** *([ADR 0005](./docs/adr/0005-git-driven-state-tracking-qa-fanout.md))*: Naive file-glob scanning of agent sandboxes breaks on real projects because Python runtimes silently write binary `.pyc` files, prior-cycle artifacts bleed into the next snapshot, and there is no scope boundary between "files the agent touched" and "files that happened to exist." Replacing glob scans with a two-primitive Git API (`init_sandbox_git` + `get_pipeline_snapshot_files`) solves all three: `.gitignore` blocks binary noise before it reaches the diff, `git diff <base_branch> --name-only` produces a strict causal delta, and the base-branch anchor is immovable so retries never accumulate stale state. This is a transferable pattern for any agentic system that writes files.

* **LLM Output Fan-Out to Break Token Ceiling** *([ADR 0005](./docs/adr/0005-git-driven-state-tracking-qa-fanout.md))*: Monolithic LLM generation of a test suite covering an entire multi-file contract hits the `max_output_tokens` wall and returns a truncated, syntactically broken file. The fix is a fan-out: one isolated LLM call per production module, results collected via `asyncio.gather`, each writing its own `test_<module_slug>.py`. This trades one large serial call for N small parallel calls — latency is bounded by the slowest module, total output tokens per call shrink to a single module's scope, and test isolation improves as a side effect. Facade modules (`__init__.py`) must be filtered before fan-out to prevent exponential coverage duplication.

* **FSM Catch-22: Architect Contract vs. Developer SOLID Compliance** *([ADR 0005](./docs/adr/0005-git-driven-state-tracking-qa-fanout.md))*: A deadlock emerges when the Developer agent (following DRY/SOLID) extracts shared logic into a utility file not listed in the architecture contract, and the Reviewer agent rejects this as a "contract violation." Neither agent can resolve the conflict within the current FSM cycle — the Architect cannot amend the contract retroactively, and the Developer cannot remove the helper without violating the design constraints. Resolution requires an explicit authorization rule in the Reviewer's system prompt: utility files created to enforce separation of concerns are pre-approved. This teaches a general principle: FSM gate agents must encode an explicit policy for authorized deviations, not just enforcement of the original contract.

* **The Python Type-Hierarchy Trap & Self-Healing** *([ADR 0004](./docs/adr/0004-modularization-sandbox-hardening.md))*: Python's type system implicitly inherits `bool` from `int` (`isinstance(True, int)` is `True`). When Developer agents generate overly complex checks such as `if not isinstance(n, int) or isinstance(n, bool):`, it introduces cognitive load and fragile logic. Utilizing an elite Reviewer agent (Gemini 2.5 Pro) to reject this code and route strict diagnostics forces the Developer agent (Claude) to align with a precise type-identity pattern: `if type(n) is not int:`.

* **Sandbox Volume Isolation & Anti-Self-Mutation** *([ADR 0004](./docs/adr/0004-modularization-sandbox-hardening.md))*: Mounting the entire working directory (`cwd`) inside execution containers exposes the pipeline's own core files to accidental or malicious modifications by run-away agent scripts. Transitioning to a strict dual-mount strategy—mounting framework code `src/` as Read-Only (`:ro`) and the volatile execution output `artifacts/` as Read-Write (`:rw`)—completely mitigates the risk of self-mutation.

* **Modularization Mandate** *([ADR 0004](./docs/adr/0004-modularization-sandbox-hardening.md))*: Monolithic orchestration (500+ lines in `orchestrator.py`) has become a primary bottleneck for maintainability and security auditing. Transitioning to a decoupled, module-based architecture (Logic/Nodes/Utils) is mandatory to mitigate path-traversal risks and allow atomic testing of pipeline components.

* **Observability Bottlenecks** *([ADR 0003](./docs/adr/0003-dual-channel-observability.md))*: Standard output (`print()`) is highly volatile and insufficient for autonomous trace tracking. A strict dual-channel logging system (Console INFO + Rotating Audit Log DEBUG) is required to ensure complete pipeline repeatability. Token counts must be extracted natively from structured responses, while out-of-band CLI executions (Claude CLI) must be audited through specialized session database parsers (`npx ccusage`).

* **LTS vs. Experimental Model Quotas** *([ADR 0003](./docs/adr/0003-dual-channel-observability.md))*: Pinning cutting-edge experimental preview models to Free Tier accounts introduces severe API bottlenecks (429 Resource Exhausted under a 20 request/day cap). For PoC development, shifting workloads to robust mainstream versions (Gemini 2.5 Flash and Pro) balances logical reasoning capacities with highly generous execution quotas.

* **Implicit Type Hazards** *([ADR 0002](./docs/adr/0002-async-qa-node-isolation.md))*: LLM reasoning consistently misses language-specific type inheritance constraints. Multi-agent code generation requires explicit runtime type-guards to counter architectural assumptions.

* **QA Boundary Isolation** *([ADR 0001](./docs/adr/0001-baseline-sequential-loop.md))*: Never grant the Developer agent write permissions to the validation scope. Without absolute environment isolation, models will mutate tests to fake a successful compilation.

* **Hybrid Model Routing & Caching** *([ADR 0000](./docs/adr/0000-cloud-infra-fsm-research.md))*: Multi-agent architectures are cost-prohibitive without strict model routing driven by tool access. Ingest heavy context via Gemini Context Caching, and bind active code mutations to Claude Code execution using ephemeral Prompt Caching to achieve up to a 90% input token discount.

* **The Custom Framework Mandate** *([ADR 0000](./docs/adr/0000-cloud-infra-fsm-research.md))*: Generic graph frameworks (e.g., LangGraph) introduce heavy dependency bloat and rigid boilerplate. Building a custom Python/Pydantic Finite State Machine (FSM) is required for absolute latency control, programmatic circuit breaking, and native visibility into execution loops.
