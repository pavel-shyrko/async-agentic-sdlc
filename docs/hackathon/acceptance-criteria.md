## 7 · Cost & Efficiency (15 pts)

> Cost & efficiency report must be present. If it doesn't exist → 0 on 7.1.
> If every agent uses the same top-tier model → **0 on 7.2, no exceptions.**

| Sub | What to check | Max |
| --- | --- | --- |
| 7.1 | Auto-collected report: tokens/cost/time **per agent role** | 4 |
| 7.2 | Different models for different roles; complex roles (Architect) → powerful model; cheap/fast tasks (formatting, log summary) → smaller model. Routing enforced in code. | 7 |
| 7.3 | Evidence of trade-off: A/B data showing where cheaper model was good enough | 4 |

### Model routing quick guide

| Role type | Expected model tier |
| --- | --- |
| Architect, Orchestrator, complex reasoning | Large / frontier |
| Developer (standard features) | Mid-tier |
| Code formatter, log summarizer, simple test gen | Small / fast |
| **All roles identical →** | **0 pts on 7.2** |

Higher total cost with same quality = **fewer points**, not more.

## 🚫 Instant Disqualification

* No written SDLC spec submitted
* Demo app was pre-built by humans
* Entire workflow is one giant LLM prompt with no agent separation
